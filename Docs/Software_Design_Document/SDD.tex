\section{Introduction}
This document explains the design of our AI-Powered University Information System.
The goal of this project is to create one platform where students can easily check their academic progress, explore campus opportunities, and communicate with advisors.
Many students currently use different websites for these tasks, so our system is meant to make the process simpler.

\section{Purpose}
The purpose of this project is to improve how students access important university information.
Instead of searching through multiple systems, students can use one application to manage their school needs.
We also want to help students plan ahead by showing progress toward graduation and useful recommendations.
In this snapshot, the system is expanded further to support stronger communication, better resource management, and more useful academic tools for both students and administrators.

\section{System Architecture}
The system follows a client-server architecture.

\textbf{Frontend:} React Native and Web Interface \\  
\textbf{Backend:} Node.js with Express \\  
\textbf{Database:} MongoDB \\  
\textbf{Authentication:} JWT-based login system

\subsection*{Existing Feature Modules}
\begin{itemize}
\item Recommendation Engine for personalized internships, scholarships, and opportunities
\item Notification System for reminders, deadlines, and campus announcements
\item Advisor Messaging Module for communication between students and advisors
\end{itemize}

\subsection*{New Snapshot 3 Feature Modules}
\begin{itemize}
\item Analytics Dashboard for tracking student progress, engagement, and opportunity activity
\item Document Upload Module for resumes, academic files, and supporting materials
\item Admin Management Module for updating opportunities, announcements, and student resources
\end{itemize}

The frontend handles the user interface, while the backend manages requests, user data, recommendations, notifications, messaging, uploaded files, and communication with the database.
The database stores student profiles, academic progress information, uploaded documents, recommendations, messages, and system announcements.

\section{User Interface}
The system includes several main pages:

\begin{itemize}
\item Login Page for secure sign in
\item Dashboard for announcements and quick access tools
\item Progress Tracker for completed units and degree progress
\item Resources Page for internships, scholarships, and events
\item Advisor Portal for advisor and student communication
\item Profile Page for account settings and personal information
\item Notifications Page for reminders, deadlines, and updates
\item Recommendation Page for personalized student opportunities
\item Analytics Dashboard Page for viewing student activity and progress summaries
\item Document Upload Page for resumes and academic file submissions
\item Admin Management Page for maintaining announcements, resources, and opportunity postings
\end{itemize}

The interface is designed to be simple and easy to navigate for students, advisors, and administrators.
The expanded interface in Snapshot 3 supports more advanced academic planning and management features while keeping the layout organized and user friendly.

\section{Snapshot 3 Goals}
For the third project snapshot, our goals are to continue expanding the system with another major set of features.
This snapshot focuses on adding analytics tools, document upload support, and administrative management functions.
These additions improve how students track their activity, how advisors review student information, and how administrators manage the overall system.
This phase also updates the design documentation to reflect the growing complexity of the application and prepares the project for final refinement in the next snapshot.
